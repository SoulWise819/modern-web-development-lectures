# todo-list-app

본 문서는 Vue.js 입문 수업에서 Todo List 앱 실습을 진행할 때 참고하는 강사용 진행 가이드이다.

## 수업 목표

- Vue 프로젝트 생성과 실행 흐름을 짧게 시연한다.
- 프로젝트 세팅 단계에서 PrimeVue 최신 버전을 함께 설치하도록 안내한다.
- Vue 핵심 기능을 기능 구현과 연결해 설명한다.
- 수강생이 상태 중심 UI 사고방식을 체감하도록 유도한다.

## 핵심 운영 원칙

- 문법을 먼저 길게 설명하지 않는다.
- 기능 하나를 구현할 때 필요한 개념만 바로 설명한다.
- 구현 후에는 반드시 상태 변화와 화면 변화의 관계를 다시 짚는다.
- 수강생이 DOM 직접 조작 관성에서 벗어나도록 질문을 던진다.
- UI는 PrimeVue를 활용해 빠르게 구성하되, 학습 초점은 Vue 상태 흐름에 둔다.

## 권장 시간 배분

- 0~15분: 프로젝트 생성, PrimeVue 설치, 파일 구조 설명
- 15~25분: 입력과 추가 기능 구현
- 25~35분: 목록 렌더링 구현
- 35~45분: 완료 처리와 빈 목록 메시지 구현
- 45~55분: 삭제 기능 구현
- 55~60분: 필터링, 남은 개수, 전체 정리

## 단계별 진행 가이드

### Part 1. 프로젝트 세팅 및 PrimeVue 설치

강의 목표

- 수강생이 Vue 프로젝트의 시작점과 UI 라이브러리 설치 흐름을 함께 이해하게 만든다.

반드시 짚을 내용

- Vue 프로젝트를 생성한 직후 의존성을 설치한다.
- PrimeVue 최신 버전과 테마 패키지를 함께 설치한다.
- 설치 권장 명령은 `npm install primevue @primevue/themes` 이다.
- `main.js` 또는 `main.ts`에서 PrimeVue를 등록한다.
- PrimeVue는 화면 작성을 빠르게 돕는 도구이고, 오늘의 핵심은 여전히 Vue 상태 관리라는 점을 분명히 한다.

강의 멘트 예시

- "오늘은 Todo 기능 자체에 집중하기 위해 UI 기본 부품은 PrimeVue 최신 버전을 사용하겠습니다."
- "버튼과 입력창을 직접 꾸미는 데 시간을 쓰기보다, Vue 상태 흐름을 이해하는 데 집중하겠습니다."
- "프로젝트는 생성만 하고 끝내지 않고, 바로 필요한 라이브러리까지 설치하는 것이 실제 개발 흐름에 가깝습니다."

주의 포인트

- PrimeVue 설치가 수업의 주제가 되면 안 된다.
- 설치 명령만 따라 치게 하지 말고 왜 넣는지 한 문장으로 설명한다.
- 테마 등록 예시는 보여 주되, 스타일 세부 설명에 시간을 과도하게 쓰지 않는다.

권장 설치 흐름

1. Vue 프로젝트 생성
2. 프로젝트 폴더 진입
3. 기본 의존성 설치
4. `npm install primevue @primevue/themes` 실행
5. `main.js` 또는 `main.ts`에 PrimeVue 등록
6. 개발 서버 실행 후 기본 화면 확인

PrimeVue 등록 예시

```ts
import { createApp } from 'vue'
import PrimeVue from 'primevue/config'
import Aura from '@primevue/themes/aura'
import App from './App.vue'

const app = createApp(App)

app.use(PrimeVue, {
  theme: {
    preset: Aura,
  },
})

app.mount('#app')
```

이번 실습에서 우선 사용할 PrimeVue 컴포넌트

- `InputText`: 할 일 입력창 구성
- `Button`: 추가, 삭제, 필터 버튼 구성
- `Checkbox`: 완료 여부 토글
- `Card`: Todo 앱 전체 영역을 한 덩어리로 정리할 때 사용 가능
- `Message`: 목록이 비었을 때 안내 문구 표시
- `Tag`: 완료, 진행 중 같은 상태 라벨 표시가 필요할 때 사용 가능

컴포넌트 선택 기준

- 처음부터 많은 컴포넌트를 쓰지 않는다.
- Todo 앱 핵심 기능 구현에 직접 필요한 것만 우선 사용한다.
- UI 컴포넌트가 늘어나도 수업의 중심은 상태, 이벤트, 렌더링 흐름이다.

권장 적용 순서

1. `InputText`, `Button`으로 입력과 추가 기능 구현
2. `Checkbox`로 완료 토글 구현
3. `Message`로 빈 목록 안내 표시
4. 필요하면 `Tag`로 상태 표시 개선
5. 마지막에 `Card`로 전체 레이아웃 정리

### Part 2. 입력과 추가 기능

연결할 개념

- `ref`
- `v-model`
- `@click`

시연 포인트

- 입력값을 상태 변수 하나에 연결한다.
- 버튼 클릭 시 항목이 배열에 추가되도록 구현한다.
- 추가 후 입력값 초기화까지 한 흐름으로 보여 준다.
- 필요하면 PrimeVue의 InputText, Button 같은 기본 컴포넌트를 사용하되 설명 포인트는 상태 변화에 둔다.

강조 메시지

- "입력창의 값을 직접 읽지 않고 상태에 연결합니다."
- "Vue에서는 이벤트가 함수로 연결되고, 함수가 상태를 바꾸면 화면이 바뀝니다."

자주 나오는 혼동

- 입력값 변수와 목록 배열의 역할을 섞는다.
- 템플릿 안에 로직을 과도하게 넣으려 한다.
- UI 컴포넌트 사용법에만 집중하고 상태 흐름을 놓친다.

질문 유도 예시

- "이 값은 화면에만 있어야 할까요, 상태로 가지고 있어야 할까요?"
- "버튼을 누를 때 실제로 무엇이 바뀌어야 하나요?"

### Part 3. 목록 렌더링

연결할 개념

- `v-for`
- `:key`

시연 포인트

- 배열의 각 항목이 화면 요소로 반복 생성되는 구조를 보여 준다.
- `key`를 왜 붙이는지 간단히 설명한다.

강조 메시지

- "우리가 반복하는 것은 HTML이 아니라 상태 데이터입니다."
- "Vue는 어떤 항목이 어떤 항목인지 알아야 안정적으로 다시 그릴 수 있습니다."

자주 나오는 혼동

- `index`를 무조건 `key`로 쓰려 한다.
- 렌더링 대상과 원본 데이터의 관계를 흐리게 이해한다.

### Part 4. 완료 처리와 조건부 표현

연결할 개념

- 클래스 바인딩
- 속성 바인딩
- `v-if`

시연 포인트

- 체크박스를 누르면 완료 상태가 바뀌고 스타일이 변하는 흐름을 보여 준다.
- 목록이 비어 있을 때만 안내 문구가 보이게 만든다.

강조 메시지

- "완료 여부는 화면 효과가 아니라 상태입니다."
- "조건문으로 DOM을 직접 다루는 대신, 보여 줄 조건을 선언합니다."

자주 나오는 혼동

- 체크박스 상태와 데이터 상태를 분리해서 생각하지 못한다.
- `v-if`와 단순 CSS 제어를 같은 것으로 이해한다.

### Part 5. 삭제 기능

연결할 개념

- 이벤트 인자 전달
- 배열 갱신

시연 포인트

- 삭제 버튼 클릭 시 특정 항목만 제거되게 구현한다.
- 삭제 후 목록이 즉시 다시 그려지는 것을 확인시킨다.

강조 메시지

- "화면 요소를 지우는 것이 아니라, 상태에서 항목을 제거하는 것입니다."

자주 나오는 혼동

- 어떤 항목을 삭제할지 식별 기준이 불명확하다.
- 배열 조작과 렌더링 결과 사이의 관계를 놓친다.

### Part 6. 필터링과 남은 개수

연결할 개념

- `computed`
- 파생 상태

시연 포인트

- 전체, 진행 중, 완료 필터를 버튼으로 전환한다.
- 남은 할 일 개수를 함께 표시한다.

강조 메시지

- "원본 목록은 그대로 두고, 보여 줄 결과만 계산합니다."
- "computed는 상태에서 파생되는 결과를 표현할 때 적합합니다."

자주 나오는 혼동

- 필터 결과를 별도 원본처럼 계속 저장하려 한다.
- 남은 개수 계산을 템플릿 안에서 과도하게 처리하려 한다.

## 수업 중 계속 던질 질문

- PrimeVue 컴포넌트를 쓰더라도 실제로 바뀌는 핵심 상태는 무엇인가
