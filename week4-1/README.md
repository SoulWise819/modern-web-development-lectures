# todo-list-app

본 문서는 Vue.js 입문 수업에서 Todo List 앱 실습을 진행할 때 참고하는 강사용 진행 가이드이다.

## 수업 목표

- Vue 프로젝트 생성과 실행 흐름을 짧게 시연한다.
- Vue 핵심 기능을 기능 구현과 연결해 설명한다.
- 수강생이 상태 중심 UI 사고방식을 체감하도록 유도한다.

## 핵심 운영 원칙

- 문법을 먼저 길게 설명하지 않는다.
- 기능 하나를 구현할 때 필요한 개념만 바로 설명한다.
- 구현 후에는 반드시 상태 변화와 화면 변화의 관계를 다시 짚는다.
- 수강생이 DOM 직접 조작 관성에서 벗어나도록 질문을 던진다.

## 권장 시간 배분

- 0~10분: 프로젝트 생성, 실행, 파일 구조 설명
- 10~20분: 입력과 추가 기능 구현
- 20~30분: 목록 렌더링 구현
- 30~40분: 완료 처리와 빈 목록 메시지 구현
- 40~50분: 삭제 기능 구현
- 50~60분: 필터링, 남은 개수, 전체 정리

## 단계별 진행 가이드

### Part 1. 프로젝트 세팅 및 파일 설명

강의 목표

- 수강생이 Vue 프로젝트가 어디서 시작되는지 이해하게 만든다.

반드시 짚을 내용

- `package.json`은 실행 명령과 의존성 정보를 담는다.
- `main.js` 또는 `main.ts`는 앱 시작점이다.
- `App.vue`는 루트 컴포넌트다.
- 처음에는 파일 전체를 설명하려 하지 말고, 지금 바로 수정할 파일 위주로 설명한다.

강의 멘트 예시

- "지금은 파일 이름을 외우는 것이 목적이 아니라, 어디서 화면이 시작되는지만 잡으면 됩니다."
- "오늘은 App.vue 중심으로만 작업하고, 필요할 때만 분리하겠습니다."

주의 포인트

- CLI나 생성 명령 자체에 시간을 너무 많이 쓰지 않는다.
- 도구 설명이 길어지면 수강생 집중이 빨리 떨어진다.

### Part 2. 입력과 추가 기능

연결할 개념

- `ref`
- `v-model`
- `@click`

시연 포인트

- 입력값을 상태 변수 하나에 연결한다.
- 버튼 클릭 시 항목이 배열에 추가되도록 구현한다.
- 추가 후 입력값 초기화까지 한 흐름으로 보여 준다.

강조 메시지

- "입력창의 값을 직접 읽지 않고 상태에 연결합니다."
- "Vue에서는 이벤트가 함수로 연결되고, 함수가 상태를 바꾸면 화면이 바뀝니다."

자주 나오는 혼동

- 입력값 변수와 목록 배열의 역할을 섞는다.
- 템플릿 안에 로직을 과도하게 넣으려 한다.

질문 유도 예시

- "이 값은 화면에만 있어야 할까요, 상태로 가지고 있어야 할까요?"
- "버튼을 누를 때 실제로 무엇이 바뀌어야 하나요?"

### Part 3. 목록 렌더링

연결할 개념

- `v-for`
- `:key`

시연 포인트

- 배열의 각 항목이 화면 요소로 반복 생성되는 구조를 보여 준다.
- `key`를 왜 붙이는지 간단히 설명한다.

강조 메시지

- "우리가 반복하는 것은 HTML이 아니라 상태 데이터입니다."
- "Vue는 어떤 항목이 어떤 항목인지 알아야 안정적으로 다시 그릴 수 있습니다."

자주 나오는 혼동

- `index`를 무조건 `key`로 쓰려 한다.
- 렌더링 대상과 원본 데이터의 관계를 흐리게 이해한다.

### Part 4. 완료 처리와 조건부 표현

연결할 개념

- 클래스 바인딩
- 속성 바인딩
- `v-if`

시연 포인트

- 체크박스를 누르면 완료 상태가 바뀌고 스타일이 변하는 흐름을 보여 준다.
- 목록이 비어 있을 때만 안내 문구가 보이게 만든다.

강조 메시지

- "완료 여부는 화면 효과가 아니라 상태입니다."
- "조건문으로 DOM을 직접 다루는 대신, 보여 줄 조건을 선언합니다."

자주 나오는 혼동

- 체크박스 상태와 데이터 상태를 분리해서 생각하지 못한다.
- `v-if`와 단순 CSS 제어를 같은 것으로 이해한다.

### Part 5. 삭제 기능

연결할 개념

- 이벤트 인자 전달
- 배열 갱신

시연 포인트

- 삭제 버튼 클릭 시 특정 항목만 제거되게 구현한다.
- 삭제 후 목록이 즉시 다시 그려지는 것을 확인시킨다.

강조 메시지

- "화면 요소를 지우는 것이 아니라, 상태에서 항목을 제거하는 것입니다."

자주 나오는 혼동

- 어떤 항목을 삭제할지 식별 기준이 불명확하다.
- 배열 조작과 렌더링 결과 사이의 관계를 놓친다.

### Part 6. 필터링과 남은 개수

연결할 개념

- `computed`
- 파생 상태

시연 포인트

- 전체, 진행 중, 완료 필터를 버튼으로 전환한다.
- 남은 할 일 개수를 함께 표시한다.

강조 메시지

- "원본 목록은 그대로 두고, 보여 줄 결과만 계산합니다."
- "computed는 상태에서 파생되는 결과를 표현할 때 적합합니다."

자주 나오는 혼동

- 필터 결과를 별도 원본처럼 계속 저장하려 한다.
- 남은 개수 계산을 템플릿 안에서 과도하게 처리하려 한다.

## 수업 중 계속 던질 질문

- 지금 바뀌어야 하는 것은 DOM인가, 상태인가
- 이 값은 원본 상태인가, 계산된 결과인가
- 지금 배우는 Vue 기능이 방금 만든 앱 기능과 어떻게 연결되는가
