# WEEK3주차 2회차 Vue Todo List 실습 가이드 (수강생용)

본 문서는 수강생이 Vue.js를 배우는 과정에서 Todo List 앱을 직접 만들어 보며 핵심 기능을 익히도록 구성한 실습 가이드이다.

## 이번 시간에 만들 것

- 할 일을 입력하고 추가하는 화면
- 추가한 할 일을 목록으로 보는 화면
- 완료 여부를 바꾸는 기능
- 항목 삭제 기능
- 전체, 진행 중, 완료 필터 기능
- 남은 할 일 개수 표시

## 이번 시간에 배우는 Vue 핵심 기능

- `ref`
- `v-model`
- `@click`
- `v-for`
- `:key`
- `:class` 또는 속성 바인딩
- `v-if`
- `computed`

## 실습 전 준비

- Node 환경이 준비되어 있어야 한다.
- Vue 공식 가이드 기준으로 새 프로젝트를 생성한다.
- 프로젝트 생성 직후 PrimeVue 최신 버전과 테마 패키지를 설치한다.
- 설치 권장 명령은 `npm install primevue @primevue/themes` 이다.
- 프로젝트 실행 후 기본 화면이 보이는지 먼저 확인한다.

## 프로젝트 세팅 단계에서 할 일

- Vue 프로젝트를 생성한다.
- 프로젝트 루트에서 의존성을 설치한다.
- PrimeVue 최신 버전과 `@primevue/themes`를 추가 설치한다.
- `main.js` 또는 `main.ts`에서 PrimeVue를 등록할 준비를 한다.
- 개발 서버를 실행해 기본 화면이 정상적으로 뜨는지 확인한다.

## 왜 PrimeVue를 먼저 설치하나

- 실습 UI를 너무 많은 CSS 작성 없이 빠르게 구성할 수 있다.
- 버튼, 입력창, 체크박스 같은 Todo 앱 기본 요소를 일관된 형태로 사용할 수 있다.
- Vue 기본 기능 학습에 집중하면서도 화면 구성 부담을 줄일 수 있다.

## 먼저 이해할 파일

- `package.json`: 실행 명령과 의존성 정보를 담는 파일
- `src/main.js` 또는 `src/main.ts`: 앱이 시작되는 파일
- `App.vue`: 현재 실습의 메인 화면을 작성할 파일
- `components/`: 필요 시 화면 조각을 나눌 폴더

PrimeVue를 사용할 경우 추가로 볼 파일

- `src/main.js` 또는 `src/main.ts`: PrimeVue 등록 위치
- 테마 preset 설정 코드: PrimeVue 스타일 적용 시작점

## 실습 순서

### 1단계. 빈 Todo 화면 준비

목표

- 앱의 제목과 입력 영역, 목록 영역이 들어갈 기본 화면을 만든다.

확인할 것

- 화면 구조가 크게 입력 영역과 목록 영역으로 나뉘는지
- 앞으로 기능을 추가할 자리가 분명한지

핵심 포인트

- 처음부터 예쁘게 만드는 것이 목적이 아니다.
- 기능을 붙이기 쉬운 구조로 시작하는 것이 더 중요하다.

### 2단계. 입력과 추가 기능 구현

목표

- 입력창에 값을 입력하고 버튼 클릭으로 할 일을 추가한다.

배우는 Vue 기능

- `ref`
- `v-model`
- `@click`

확인할 것

- 입력값이 상태에 연결되는지
- 버튼을 누르면 새 항목이 추가되는지
- 추가 후 입력창이 비워지는지

스스로 점검할 질문

- 왜 입력값을 직접 DOM에서 읽지 않아도 되는가
- 왜 추가 로직은 함수로 분리하는가

### 3단계. 목록 렌더링

목표

- 추가한 할 일을 화면에 반복 목록으로 출력한다.

배우는 Vue 기능

- `v-for`
- `:key`

확인할 것

- 여러 항목이 목록으로 잘 보이는지
- 각 항목이 고유하게 구분되는지

스스로 점검할 질문

- 배열 상태가 바뀌면 화면이 왜 갱신되는가
- `key`는 왜 필요한가

### 4단계. 완료 처리와 빈 목록 메시지

목표

- 각 항목의 완료 여부를 바꾸고, 목록이 없을 때 안내 문구를 보여 준다.

배우는 Vue 기능

- 속성 바인딩
- 클래스 바인딩
- `v-if`

확인할 것

- 체크 시 완료 상태가 바뀌는지
- 완료된 항목의 스타일이 달라지는지
- 목록이 비어 있으면 안내 문구가 보이는지

스스로 점검할 질문

- 완료 상태를 어떻게 화면에 표현할 수 있는가
- `v-if`는 어떤 상황에서 유용한가

### 5단계. 삭제 기능 구현

목표

- 특정 할 일을 선택해서 제거한다.

배우는 Vue 기능

- 이벤트 인자 전달
- 배열 갱신

확인할 것

- 삭제 버튼을 누르면 해당 항목만 제거되는지
- 다른 항목에는 영향이 없는지

스스로 점검할 질문

- 어떤 기준으로 삭제할 항목을 찾는가
- 화면을 지우는 것이 아니라 상태를 바꾸는 이유는 무엇인가

### 6단계. 필터링과 남은 개수 표시

#### 목표

- 전체, 진행 중, 완료 상태에 따라 목록을 필터링한다.
- 남은 할 일 개수를 표시한다.

#### 배우는 Vue 기능

- `computed`
- 파생 상태

#### 확인할 것

- 버튼 클릭에 따라 목록이 바뀌는지
- 남은 개수가 정확히 표시되는지

#### 스스로 점검할 질문

- 필터된 목록은 원본 데이터와 어떻게 다른가
- 남은 개수는 왜 계산된 값으로 다루는 편이 좋은가

## 최종 체크리스트

- 입력이 가능하다.
- 새 할 일을 추가할 수 있다.
- 목록이 정상적으로 출력된다.
- 완료 여부를 바꿀 수 있다.
- 항목을 삭제할 수 있다.
- 빈 목록 메시지가 보인다.
- 필터 버튼이 동작한다.
- 남은 할 일 개수가 맞다.

## 이번 실습의 핵심 정리

- Vue는 DOM을 직접 조작하기보다 상태를 바꾸면 화면이 반응하는 구조를 사용한다.
- Todo List 앱은 작은 예제이지만 Vue 핵심 기능을 실제로 연결해서 이해하기 좋다.
- 중요한 것은 문법 암기보다 상태를 어떻게 설계하고 화면과 연결하는지 이해하는 것이다.

## 실습 후 확장 과제

- 항목 수정 기능 추가
- 완료 항목 일괄 삭제
- 로컬 스토리지 저장
- 컴포넌트 분리
- 입력값 검증 추가
